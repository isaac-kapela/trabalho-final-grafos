REQUISITOS:
-----------
- Compilador g++ com suporte a C++17
- Sistema Linux
- Make instalado


PASSO 1: COMPILAR O PROJETO
----------------------------

No diretório raiz do projeto (/home/oid100739/ufjf/trabalho-final-grafos):

    make

Isso irá:
- Criar os diretórios necessários (build/, bin/, results/, instances/)
- Compilar todos os arquivos fonte
- Gerar o executável em: bin/dcmst


PASSO 2: TESTAR A INFRAESTRUTURA
---------------------------------

Execute o teste automático da infraestrutura:

    ./bin/dcmst teste

Ou usando o Makefile:

    make teste

Este teste irá verificar:
- Geração de números aleatórios com semente
- Criação e manipulação de grafos
- Geração de logs em CSV
- Exportação para visualização em Graph Editor

Arquivos gerados em results/:
- log_teste.csv           (log da execução)
- teste_grafo.txt         (grafo para visualização)


PASSO 3: TESTAR COM DIFERENTES INSTÂNCIAS
------------------------------------------

Você tem duas opções para testar o programa:

A INSTÂNCIAS OR-LIBRARY 
---------------------------------------

O projeto contém instâncias oficiais da OR-Library em dcmst/Data/:

    ./bin/dcmst orlib dcmst/Data crd101 3

Instâncias disponíveis:
- crd101, crd103, crd105, crd108, crd109, crd500, crd700, crd705
- rand200
- str305, str505, str509, str701, str702, str1001, str1005
- sym306, sym708
- shrd259
- genhard

Formato do comando:
    ./bin/dcmst orlib <diretorio> <nome_instancia> <grau_maximo>

O formato OR-Library usa coordenadas (x,y) e calcula distâncias euclidianas
para criar um grafo completo automaticamente.

Exemplos práticos com OR-Library:
    ./bin/dcmst orlib dcmst/Data crd103 5
    ./bin/dcmst orlib dcmst/Data rand200 4
    ./bin/dcmst orlib dcmst/Data str305 5


B) INSTÂNCIAS CUSTOMIZADAS (OPCIONAL)
--------------------------------------

Para testar com suas próprias instâncias no formato padrão:

    ./bin/dcmst ler instances/exemplo.txt

NOTA: Você precisa criar o arquivo instances/exemplo.txt primeiro!


FORMATO DAS INSTÂNCIAS CUSTOMIZADAS
-----------------------------------

As instâncias customizadas devem seguir o formato:

    <num_vertices> <num_arestas> <grau_maximo>
    <origem> <destino> <peso>
    <origem> <destino> <peso>
    ...

IMPORTANTE: Os vértices são numerados a partir de 1.

Exemplo (instances/exemplo.txt):
    5 7 3
    1 2 1.5
    2 3 2.0
    3 4 1.0
    4 5 3.0
    5 1 2.5
    1 3 4.0
    2 5 1.8


COMANDOS ÚTEIS
--------------

Compilar:
    make

Limpar arquivos compilados:
    make clean

Limpar tudo (compilados + resultados):
    make cleanall

Testar infraestrutura:
    make teste

Ver ajuda do Makefile:
    make help

Executar com comando específico:
    ./bin/dcmst <comando> [parâmetros]

Comandos disponíveis:
    teste              - Testa a infraestrutura básica
    ler <arquivo>      - Lê uma instância customizada
    orlib <diretorio> <nome_instancia> <grau_max>  - Lê instância OR-Library (recomendado)

Exemplo de uso das instâncias OR-Library:
    ./bin/dcmst orlib dcmst/Data crd101 3
    ./bin/dcmst orlib dcmst/Data rand200 5
    ./bin/dcmst orlib dcmst/Data str305 4

SOLUÇÃO DE PROBLEMAS
--------------------

Erro ao compilar:
- Verifique se g++ está instalado: g++ --version
- Verifique se está no diretório correto
- Execute: make clean && make

Erro ao executar:
- Verifique se compilou com sucesso
- Verifique permissões: chmod +x bin/dcmst
- Execute a partir do diretório raiz do projeto

Arquivo de instância não encontrado:
- Para instâncias OR-Library: use dcmst/Data <nome_instancia>
  Exemplo: ./bin/dcmst orlib dcmst/Data crd101 3
  
- Para suas próprias instâncias: use instances/<nome_arquivo>
  Exemplo: ./bin/dcmst ler instances/exemplo.txt
  
- Verifique se o arquivo realmente existe no diretório correto
- Use caminho relativo ou absoluto completo

Qual formato usar?
- Use "orlib" para instâncias oficiais do OR-Library (formato coordenadas)
- Use "ler" para suas instâncias customizadas (formato lista de arestas)
